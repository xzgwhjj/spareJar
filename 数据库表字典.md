> 本文档基于 `uniCloud-alipay/database/` 下全部 31 个集合的 `*.schema.json` 与 `*.index.json` 自动提取生成，并整合了《余钱罐.md》§8.9「核心数据模型」与 §12.6「皮肤数据模型」的既有描述。
>
> - 生成日期：2026-07-17
> - 集合总数：**31**
> - 数据来源：`uniCloud-alipay/database/*.schema.json`（结构）、`*.index.json`（索引）、`db_init.json`（初始数据）
> - 说明：规划文档《余钱罐.md》§8.9 仅列出 10 个核心集合（规划视角），实际落库为 31 个集合，本文档以**实际 schema 为准**。

## 一、全局约定

| 约定项 | 说明 |
| --- | --- |
| 主键 `_id` | 所有集合默认由数据库自动生成 ObjectId；仅 `uni-id-users._id` 使用微信 openid 作为主键。 |
| 业务主键 `user_id` | 值为**微信 openid**，与 `users.user_id`、uni-id 的 `uid` 一致。绝大多数业务表通过 `user_id` 归属用户。 |
| 金额单位 | 所有金额字段单位为**分（整数）**，字段描述中标注「(分)」；热量单位为 kcal。 |
| 软删除 | 可删除的业务表以 `deleted_at`（timestamp）标记删除，查询需自行过滤。 |
| 时间字段 | `created_at` / `updated_at` 为 timestamp；`date_key` 为 `YYYY-MM-DD` 字符串，`month_key` 为 `YYYY-MM` 字符串。 |
| 权限模型 | 业务表 `read` 规则多为 `doc.user_id == auth.uid`；日志/结算类表 `create/update/delete` 均为 `false`（仅服务端写入）；全局配置表（`achievements`、`jar_skins`）`read=true` 且不可写。 |
| 虚拟资金边界 | 存款池、心愿、惩罚扣除等均为**记账层虚拟额度**，不涉及真实资金流转（见《余钱罐.md》§8 优化建议）。 |

# 余钱罐 · 全功能开发计划

> 基于 `余钱罐.md` 产品全案制定。覆盖文档每一个功能点（不删减、不优先级筛选），按功能依赖关系排序，采用「功能先行、样式并行迭代」策略，确保闭环完成全部功能开发。

## 〇、执行总策略

1. **功能先行、样式并行迭代**：每个阶段先打通「数据模型 → 云函数/逻辑 → 页面骨架与可用交互」（功能闭环），同时在同阶段并行补齐「主题变量、布局、玻璃拟态、动画、响应式」等样式细节；样式不阻塞功能提交，但随功能同一迭代内验收。
2. **依赖驱动排序**：以「账号基座 → 记账核心 → 限额引擎 → 首页聚合 → 攒钱/挑战闭环 → 贴纸/OCR → 资产账户 → 餐次热量 → 皮肤 → 非功能上线」为拓扑顺序；文档中的 P0/P1/P2 与 V1.0/V1.5/V2.0 标签在各阶段内以 `(Vx.x)` 标注，**全部纳入计划不省略**。
3. **数据模型先行**：阶段 0 一次性建好 §8.9 全部集合，避免跨版本大改表结构。
